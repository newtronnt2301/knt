# KNT Classroom — สถานะงานและแนวทางทำต่อ

อัปเดตล่าสุด: 18 กรกฎาคม 2569

## เป้าหมายของโปรเจกต์

สร้าง KNT Classroom ใหม่ภายในโฟลเดอร์ `app/` โดยใช้ดีไซน์ Soft Gradient, Mobile First และรองรับ PWA โดยห้ามแก้ไขหรือลบหน้าเว็บเดิมระหว่างการพัฒนา

แอปต้องมีสองบทบาทในแอปเดียวกัน:

- นักเรียน: หน้าแรก รายวิชา ข้อสอบ งาน และโปรไฟล์
- ครู: ภาพรวม เช็กชื่อ งาน/สแกน คะแนน เครื่องมือ และโปรไฟล์ครู

เป้าหมายสำคัญของฝั่งครูคือ “ย้ายการทำงานเข้ามาเป็นหน้าแอปจริง” ไม่ใช่เพียงทำการ์ดลิงก์, เปิดหน้าเว็บเดิม, หรือฝังหน้าเดิมด้วย iframe

## ไฟล์ที่สร้างแล้ว

- `app/index.html` — โครงหน้าแอป ไอคอน และตัวสลับบทบาท
- `app/styles.css` — Soft Gradient และ Responsive Layout
- `app/app.js` — SPA routes, หน้านักเรียน, หน้าครู และโมดูลเครื่องมือ
- `app/roster-data.js` — สำเนารายชื่อนักเรียนแบบอ่านอย่างเดียว 29 ห้อง รวม 1,173 คน เพื่อให้เปิดผ่าน `file://` ได้
- `app/manifest.webmanifest` — PWA manifest และ shortcuts สำหรับนักเรียน/ครู
- `app/sw.js` — Service Worker ปัจจุบันใช้ cache `knt-classroom-v7`
- `app/icons/` — ไอคอน PWA ขนาด 192, 512 และ SVG

ไฟล์เดิมนอก `app/` ยังไม่ได้ถูกแก้ไขจากงาน KNT Classroom รอบนี้

## ส่วนที่ทำเสร็จแล้ว

### เช็กชื่อในแอป (18 กรกฎาคม 2569)

- แยกระบบเช็กชื่อรวมออกจากเช็กชื่อเตรียมวิศวะชัดเจน
- เช็กชื่อรวมต้องเลือกครูผู้สอน, รายวิชา, ห้อง และวันที่ก่อนบันทึก
- เช็กชื่อเตรียมวิศวะล็อกให้เลือกได้เฉพาะ ม.4/1, ม.5/1 และ ม.6/1
- การบันทึกเตรียมวิศวะส่งรหัสห้องและรูปแบบสถานะให้ตรงกับ Google Apps Script เดิม
- เพิ่มปุ่มประวัติในทั้งสองระบบ แสดงวัน ห้อง ครู/วิชา และสรุปสถานะจากรายการที่บันทึกในอุปกรณ์
- ประวัติเตรียมวิศวะโหลดรายการเก่าจาก Google Sheets ของทั้ง 3 ห้อง แล้วรวมกับรายการใหม่ในอุปกรณ์

### โหมดนักเรียน

- หน้าแรก
- เมนูล่าง 5 รายการ
- หน้ารายวิชา
- หน้าข้อสอบ
- หน้างาน
- หน้าโปรไฟล์และการตั้งค่า
- ข้อมูลตัวอย่างและตัวกรองพื้นฐาน

### โหมดครู

- สลับบทบาทนักเรียน/ครูในแอปเดียวกัน
- Teacher Dashboard
- เมนูครู: ภาพรวม, เช็กชื่อ, งาน/สแกน, คะแนน, เครื่องมือ
- คลังเครื่องมือ 13 รายการจากเว็บเดิม
- ถอด iframe และลิงก์เปิดหน้าเว็บเดิมออกจากการเปิดเครื่องมือหลักแล้ว
- ทุกเครื่องมือเปิดเป็น route `#tool` และใช้หน้าตา Soft Gradient ของ KNT Classroom

### การเชื่อมข้อมูลที่ทดสอบแล้ว

- เช็กชื่อรายวิชาโหลดช่องรายวิชาจาก Google Sheets ได้ 10 ช่อง
- โหลดห้องเรียนจากรายชื่อที่ฝังในแอปได้ 29 ห้อง
- ห้อง ม.6/1 แสดงนักเรียนจริง 36 คน
- สถานะ มา/สาย/ลา/ขาด ทำงานและสรุปจำนวนได้
- ผลการเรียนโหลดรายการชุดข้อมูลจริงได้ 4 ชุด
- เปิดตารางผลการเรียนจริงได้ 36 คน และ 15 คอลัมน์
- รายงานข้อสอบโหลดข้อมูลล่าสุดได้ 100 รายการ
- ระบบส่งงานโหลดชุดห้อง/วิชาจริงได้ 6 ชุด
- ตารางกว้างเลื่อนภายในกรอบได้โดยไม่ทำให้หน้ามือถือล้น

### ระบบติดตามงานที่ย้ายเข้าแอปแล้ว (18 กรกฎาคม 2569)

- สร้าง แก้ชื่อ และลบห้อง/รายวิชา
- เพิ่ม แก้ไข ลบ และนำเข้านักเรียนจากทะเบียน Excel, CSV หรือ JSON
- เพิ่ม แก้ชื่อ และลบงาน/ข้อสอบ
- กดสถานะส่งแล้ว/ยังไม่ส่ง และกรอกคะแนนข้อสอบ
- บันทึกร่างลง `localStorage` โดยไม่เขียนทับคลาวด์อัตโนมัติ
- กล่องรหัสครูและปุ่มบันทึก Google Sheets ใช้รูปแบบ request เดียวกับระบบเดิม และมีการยืนยันก่อนเขียนทับ
- สแกน QR ด้วยกล้อง หรือกรอกรหัส/เลขที่แทนกรณีใช้กล้องไม่ได้
- สแกนเพื่อบันทึกสถานะงานหรือคะแนน พร้อมประวัติภายในรอบสแกน
- รายงานอัตราส่ง จำนวนงานค้าง และรายชื่อรายคน
- ส่งออกตารางเป็น `.xlsx`
- ทดสอบกับข้อมูลจริง 6 ชุด: ห้อง ม.4/2 แสดง 42 คน 11 งาน ตาราง 42 แถว และรายงานงานค้างได้
- ตรวจขนาด 390×844 และ 1440×900 แล้ว หน้าไม่ล้นแนวนอน

## จุดเชื่อมระบบเดิม

ค่าปัจจุบันอยู่ใน `app/app.js` ภายใต้ตัวแปร `API`

- เช็กชื่อรายวิชา: Google Apps Script ของ `เช็คชื่อรายวิชา/index.html`
- เช็กชื่อเตรียมวิศวะและผลการเรียน: Google Apps Script ชุดกลาง
- ติดตามส่งงาน: Google Apps Script ของ `ส่งงาน/index-ส่งงาน.html`
- ข้อสอบ: Google Apps Script ของ `exam/index.html`
- เงินห้อง: Google Apps Script ของ `เก็บเงินเพื่อน/`

ห้ามเปลี่ยน URL หรือรูปแบบ request โดยเดาเอง ต้องอ่านการเรียกใช้งานจากหน้าเดิมก่อนทุกครั้ง

## ข้อเท็จจริงสำคัญ: สิ่งที่ยังต้องทดสอบก่อนถือว่าสมบูรณ์

ฟังก์ชันหลักถูกย้ายเข้า `app/` แล้ว แต่ยังไม่ควรเรียกว่าสมบูรณ์จนกว่าจะผ่านรายการต่อไปนี้:

- ใช้รหัสครูจริงบันทึกห้องทดสอบขึ้น Google Sheets แล้วโหลดซ้ำว่าข้อมูลยังอยู่
- ทดสอบกล้อง QR บนโทรศัพท์จริงผ่าน HTTPS
- ดาวน์โหลดและเปิดไฟล์ Excel ที่ส่งออกจริง
- เพิ่มส่งออกรายงานเป็นภาพสำหรับแชร์หากยังต้องใช้งานแบบเดียวกับหน้าเดิม

## Source of truth สำหรับระบบติดตามงาน

อ่านไฟล์นี้ก่อนเริ่มทำต่อ:

- `ส่งงาน/index-ส่งงาน.html`

ส่วนสำคัญในไฟล์เดิม:

- `CLOUD_URL` — URL คลาวด์หลัก
- `SAVE_PW='2301'` — รหัสปลดล็อกบันทึก
- `load()` และ `save()` — local cache และ cloud save
- `refreshFromCloud()` — โหลดข้อมูลล่าสุด
- `renderSetBar()` และ `openSet()` — ห้อง/วิชา
- ส่วน render ตารางนักเรียนและชิ้นงาน
- dialog สร้างห้องและนำเข้ารายชื่อ
- `Html5Qrcode` และฟังก์ชัน scanner ช่วงท้ายไฟล์
- ส่วน export Excel, ภาพ และรายงาน

โครงข้อมูลหลักของระบบเดิมอยู่ในตัวแปร `DB` และมีรูปแบบโดยย่อ:

```text
DB.sets[key]
  room
  subject
  students[]
  works[]
  data[studentId][workId]
```

ก่อนเขียนข้อมูลกลับ ต้องรักษาโครงสร้างนี้ให้เหมือนระบบเดิมทุกประการ

## ลำดับงานแนะนำสำหรับ session ถัดไป

1. อ่าน `KNT-CLASSROOM-STATUS.md`
2. สร้างห้องทดสอบขนาดเล็ก แล้วใช้รหัสครูบันทึกขึ้น Google Sheets
3. โหลดจากคลาวด์ซ้ำ แล้วตรวจห้อง นักเรียน งาน สถานะ และคะแนน
4. ทดสอบสแกนการ์ด QR บนมือถือผ่าน HTTPS ทั้งงานส่งและข้อสอบ
5. ส่งออก Excel และเปิดตรวจคอลัมน์ สถานะ คะแนน และภาษาไทย
6. เพิ่มการส่งออกภาพรายงาน หากครูยังต้องแชร์เข้า LINE ในรูปแบบภาพ

## เกณฑ์ถือว่าระบบติดตามงานเสร็จ

ต้องทดสอบผ่านอย่างน้อยดังนี้:

- โหลดห้องจริงจากคลาวด์
- เปิดห้องและเห็นนักเรียน/งานครบ
- สร้างห้องทดสอบ
- เพิ่มนักเรียนและชิ้นงาน
- เปลี่ยนสถานะและกรอกคะแนน
- บันทึกขึ้นคลาวด์แล้วโหลดซ้ำข้อมูลยังอยู่
- สแกน QR แล้วระบุนักเรียนถูกต้อง
- สแกนเพื่อส่งงานและให้คะแนนได้
- Export Excel เปิดได้และข้อมูลถูกต้อง
- ใช้งานบนมือถือโดยไม่มีแนวนอนล้น
